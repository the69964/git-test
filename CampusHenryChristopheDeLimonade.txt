 Documentation du Système de Gestion Hospitalière 
  
Résumé 
Ce projet présente un système de gestion hospitalière permettant d’enregistrer, de modifier et de consulter les données des patients, médecins, consultations et ordonnances. 
L'algorithme est structuré de manière modulaire avec des fonctions pour la gestion, la recherche, la modification et la validation des données. 
  
 Structures de Données 
•	Patients : Identifiant, nom, prénom, sexe, âge, adresse, téléphone, groupe sanguin. 
•	Médecins : Identifiant, nom, prénom, sexe, adresse, téléphone, salaire. 
•	Consultations : Identifiant consultation, date, identifiant patient, identifiant médecin, motif. 
•	Ordonnances : Identifiant patient, identifiant consultation, médicament, posologie. 
•	Tableaux utilisés : Pat[100], Med[50], Cons[200], Ord[200] 
•	Variables globales : nbrePat, nbreMed, nbreCons, nbreOrd (compteurs) 
  
Fonctionnalités Principales Algorithme Principal 
• 	Procédure Principal : Affiche le menu principal et oriente vers les modules de gestion jusqu’à ce que l’utilisateur quitte le programme. 
  
Gestion des Patients 
•	EnregistrerPatient : Ajoute un patient avec vérification de l’unicité de l’identifiant et validité des champs. 
•	ListerPatients : Affiche la liste complète des patients enregistrés. 
•	RechercherPatient : Recherche un patient par identifiant. 
•	ModifierPatient : Met à jour les informations d’un patient existant. 
  
Gestion des Médecins 
•	EnregistrerMedecin : Ajoute un médecin avec vérification. 
•	ListerMedecins : Affiche tous les médecins enregistrés. 
•	RechercherMedecin : Recherche un médecin par identifiant. 
•	ModifierMedecin : Met à jour les informations d’un médecin. 
•	RevoquerMedecin : Supprime un médecin du système. 
•	AugmenterSalaire : Augmente le salaire de tous les médecins de 15%. 
  
Gestion des Consultations 
•	PlanifierConsultation : Crée une nouvelle consultation après validation des identifiants patient et médecin. 
•	ListerToutesConsultations : Affiche toutes les consultations. 
•	ConsultationsParDate : Affiche les consultations pour une date spécifique. 
•	ConsultationsParMedecin : Affiche les consultations d’un médecin donné. 
  
Gestion des Ordonnances 
•	AjouterOrdonnance : Ajoute une ordonnance liée à un patient et une consultation existante. 
•	AfficherOrdonnancePatient : Affiche toutes les ordonnances d’un patient donné. 
  
 Fonctions Utilitaires 
•	TestChampVide : Vérifie si un champ est vide. 
•	TestIdPatient / TestIdMedecin / TestIdConsultation : Vérifient si un identifiant est déjà utilisé. 
•	TestIdPatientExistant / TestIdMedecinExistant / TestIdConsultationExistant : Vérifient si un identifiant existe dans la structure. 
  
 Variables Globales 
•	Pat, Med, Cons, Ord : Tableaux contenant les données. 
•	nbrePat, nbreMed, nbreCons, nbreOrd : Compteurs pour chaque entité.
